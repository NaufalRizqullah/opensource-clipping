Privacy Policy

App: Automation Clippers (Automation_Clppers) Effective: July 20, 2026

1. Overview

Automation Clippers (referred to as the "Application") is an open-source, internal content administration tool operating under the Opensource Clipping initiative. The Application connects to the Meta Graph API to facilitate structured media uploads, scheduling, and publishing for Facebook Pages owned or operated by authorized system administrators.

This Privacy Policy is designed to clearly specify how the Application interacts with Meta API tokens, what platform metrics or data points are processed, and the direct methodologies in place to request absolute data deletion.

2. Information Accessed & Processed

To operate the content publishing flows securely, the Application requests and processes the following data attributes via Meta API interfaces, subject to permissions authorized by the system administrator:

  • Page Metadata: Facebook Page titles, identifiers, and configuration profiles.
  • Access Tokens: Temporary or long-lived Facebook Page access tokens needed to perform write actions on behalf of the administrator.
  • Media Content: Asset links, video streams, image objects, and status text descriptions prepared for scheduling.
  • Publishing Statuses: API transaction receipts, post identifiers, and processing success flags.
  • Diagnostics: Error logs and runtime trace information to debug API rate limits and structural errors.

3. Usage of Processed Information

All platform credentials, content metadata, and API responses are processed exclusively to execute the operations requested by the administrator:

  • Verifying the administrator's authorization boundaries across target Pages.
  • Transmitting structured media objects directly to Meta Graph API media endpoints.
  • Monitoring async media processing queues (such as Facebook Video processing) and returning receipts.
  • Diagnosing application performance bottlenecks and API validation failures.

4. Storage & Security

Page identifiers, access credentials, publishing logs, and runtime configurations are stored within private, secure storage environments configured by the deployment administrator.

Access tokens are stored using industry-standard security frameworks. Under no circumstances are active access tokens, private keys, or client secrets exposed in public source control systems, public logs, or to unauthorized client applications.

5. Data Sharing & Commercial Disclosure

We do not monetize, sell, trade, or share user information, Page assets, token hashes, or publishing logs with third-party advertisers, data brokers, or external entities.

Information is shared only with Meta Platforms, Inc. (to perform the core posting tasks) or infrastructure providers hosting the Application (e.g. database, cloud hosting), and when required by legal processes.

6. Data Retention Boundaries

We retain data records only as long as is necessary to satisfy operational logs, scheduling pipelines, and debugging histories. If an administrator detaches their Facebook account or deletes a Page connection, all corresponding local tokens and cached metadata will be permanently expunged from the application storage in a timely manner.

7. User Data Deletion Instructions

In compliance with Meta Platform Terms, users and administrators have the right to request deletion of any cached information, logs, or credentials maintained by the Application.

Official Data Deletion Request Process

To request the immediate and permanent deletion of access tokens, cached metadata, Page records, or transaction logs related to your account:

Contact Email: mereskaai@gmail.com
Subject Line: Automation_Clppers Data Deletion Request

Required Information:

When submitting a deletion request, please include the following details so we can locate your information:

  • Your administrative username or profile identifier.
  • The specific Page Names or Page IDs connected to the Application.
  • A clear list of elements you want removed (e.g. "Delete all access tokens", "Clear scheduling history", or "Wipe all profile data").

Upon receipt and verification of your email request, we will delete all associated API secrets, keys, identifiers, and activity records within five (5) business days. Confirmation will be sent back to your contact address once the deletion process completes.

Note: You can revoke the Application's authority directly through your Facebook Business Integration Settings at any time. Revoking permissions stops any future data retrieval but will not automatically wipe data stored previously; to remove cached data completely, please proceed with the email request above.

8. Changes to This Policy

This Privacy Policy may be modified periodically to reflect updates in Facebook API guidelines, Meta compliance protocols, or operational refactoring. Changes take effect immediately upon their publication on this page.
